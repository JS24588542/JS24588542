    SpringBoot后端搭建

    发表于 2023-12-24 更新于 2024-01-07 分类于 后端 本文字数： 5k 阅读时长 ≈ 5 分钟

    SpringBoot后端搭建过程，以centos7+docker24云服务器为操作对象。

    部署数据库

    这里以MySQL为例

    部署MySQL数据库

    1. 拉取MySQL镜像：

      sudo docker pull mysql

    2. 运行MySQL容器：

      sudo docker run --name mysql-server -e MYSQL_ROOT_PASSWORD=yourpassword -d mysql

    3. 查看容器运行状态：

      sudo docker ps

    4. 进入MySQL容器的命令行

      sudo docker exec -it mysql-server mysql -uroot -pyourpassword

    5. 现在可以使用SQL语句来操作了。

    安装Java环境

    这里以Java 17为例

    1. 下载Java 17 二进制文件

      wget https://download.oracle.com/java/17/latest/jdk-17_linux-x64_bin.rpm

    2. 安装

      yum -y install ./jdk-17_linux-x64_bin.rpm

    3. 验证

      java -version

    使用 Navicat 生成和导入数据

    步骤 1: 创建 MySQL 数据库和用户表

    1. 打开 Navicat 并连接到你的 MySQL 实例。
    2. 创建新数据库：
      • 右键单击连接名称，选择 新建数据库。
      • 命名数据库为 db。
    3. 创建用户表：
      • 在 db 数据库上右键，选择 新建表。
      • 定义表的结构（例如：id, name, email）。

    步骤 2: 生成测试数据

    1. 打开用户表：
      • 双击刚刚创建的用户表。
    2. 插入数据：
      • 可以手动添加数据，或使用 Navicat 的 生成数据 功能自动填充测试数据。

    步骤 3: 导出数据

    1. 右键用户表，选择 转储 SQL 文件 -> 结构和数据。
    2. 保存 SQL 文件：
      • 选择保存位置并保存导出的 SQL 文件。

    步骤 4: 传输数据到服务器

    1. 使用 SCP、FTP 或其他文件传输方法将 SQL 文件传输到服务器。

      scp /path/to/local/sqlfile.sql user@your_server:/path/to/remote/

    步骤 5: 在服务器的 Docker MySQL 容器中使用数据

    1. 登录到你的服务器。

    2. 复制 SQL 文件到 Docker 容器：

      docker cp /path/to/remote/sqlfile.sql your_mysql_container:/tmp/sqlfile.sql

    3. 导入数据到 Docker 容器中的 MySQL：

      use db;
source db.sql

      有一堆的ok就ok

    4. 注意事项

    • 确保在导出和导入数据时使用相同的字符集，以避免编码问题。
    • 在导入数据前，确认数据库 db 在 Docker 容器中已存在，或在导入 SQL 文件时创建它。
    • 如果服务器配置了防火墙或安全组，请确保相应的端口（通常是22端口用于SSH）开放以便于文件传输。
    • 在使用任何敏感数据前，确保遵守相关的数据保护和隐私法规。

    安装Minio

    1. 拉取镜像

      docker pull minio/minio

    2. 创建目录
      一个用来存放配置，一个用来存储上传文件的目录
      启动前需要先创建Minio外部挂载的配置文件（ /home/minio/config）,和存储上传文件的目录（ /home/minio/data）

      mkdir -p /home/minio/config
mkdir -p /home/minio/data

    3. 创建Minio容器并运行

      9090端口指的是minio的客户端端口
      MINIO_ACCESS_KEY ：账号
      MINIO_SECRET_KEY ：密码（账号长度必须大于等于5，密码长度必须大于等于8位）

      • 使用宿主机网络模式:

        docker run --net=host \
--name minio \
-d --restart=always \
-e "MINIO_ACCESS_KEY=minioadmin" \
-e "MINIO_SECRET_KEY=minioadmin" \
-v /home/minio/data:/data \
-v /home/minio/config:/root/.minio \
minio/minio server /data --console-address ":9090" --address ":9000"

      • 不使用宿主机网络模式:

        docker run -p 9000:9000 -p 9090:9090 \
--name minio \
-d --restart=always \
-e "MINIO_ACCESS_KEY=minioadmin" \
-e "MINIO_SECRET_KEY=minioadmin" \
-v /home/minio/data:/data \
-v /home/minio/config:/root/.minio \
minio/minio server /data --console-address ":9090" --address ":9000"

      注：如何选择？
      当在 Docker 中部署应用程序时，选择使用宿主机网络模式 (–net=host) 还是标准的端口映射模式，取决于你的具体需求和环境。这两种模式各有优缺点：

      • 宿主机网络模式 (–net=host)
        • 性能:
          • 提供更好的网络性能，因为它避免了 Docker 的网络堆栈，直接使用宿主机的网络。
          • 减少了网络延迟。
        • 简化配置:
          • 不需要端口映射，容器可以直接打开和使用宿主机的端口。
        • 安全性和隔离:
          • 减少了网络隔离，容器可以访问宿主机上的所有网络接口。
          • 可能会有安全风险，特别是在多容器环境下。
        • 使用场景:
          • 适用于需要高性能网络或需要访问宿主机网络配置的应用。
          • 在单容器运行环境或安全可控的环境中更常见。
      • 标准的端口映射模式
        • 网络隔离:
          • 提供了更好的网络隔离，每个容器都有自己的网络命名空间。
          • 安全性较高，因为它限制了容器可以访问的网络资源。
        • 端口映射:
          • 需要手动设置端口映射，这可以使得同一宿主机上的多个容器共存。
        • 灵活性和通用性:
          • 更加灵活和通用，适用于大多数应用场景。
          • 容易与 Docker 网络生态系统（如 Docker Compose）集成。
        • 性能:
          • 可能会有轻微的性能损耗，因为网络流量需要通过 Docker 的网络堆栈。
      • 如何选择？
        • 如果你需要最大化网络性能，或者容器需要使用某些特定的宿主机网络配置（例如，使用宿主机的 IP 地址），可以考虑使用宿主机网络模式。
        • 如果你更注重安全性、网络隔离或者在同一宿主机上运行多个容器，建议使用标准的端口映射模式。
        • 在大多数情况下，标准的端口映射模式是更安全、更易于管理的选择，特别是在多容器环境中。然而，对于一些特定的用例（例如高性能计算或网络功能），宿主机网络模式可能是更合适的选择。

    开放防火墙

    1. 到云服务器控制台开放端口

    2. 启动Firewalld

      sudo systemctl start firewalld
sudo systemctl enable firewalld

    3. 开放宿主机防火墙规则

      
# 允许 9000 端口的流量
 sudo firewall-cmd --zone=public --add-port=9000/tcp --permanent

 # 允许 9090 端口的流量
 sudo firewall-cmd --zone=public --add-port=9090/tcp --permanent

 # 重新加载防火墙规则使更改生效
 sudo firewall-cmd --reload

    构建SpringBoot项目

    1. 使用Spring Initializer来完成构建

    123

    在使用 MyBatis 与 Spring Boot 结合时，应用 DAO 和 DO 模式稍有不同。MyBatis 是一个半自动的 ORM（对象关系映射）框架，它允许更精细的控制 SQL 语句，同时提供了映射实体到数据库记录的能力。

    1. 创建 DO（或 DTO）
      首先定义 DO（Data Object）或 DTO（Data Transfer Object），它们用于表示数据库表的数据结构。

    java
    Copy code
    // 例：User.java
    public class User {
    private Long id;
    private String name;
    private String email;
    // 省略getter和setter方法
    }
    2. 定义 MyBatis Mapper 接口
    在 MyBatis 中，你需要定义一个 Mapper 接口，它相当于传统意义上的 DAO 接口。

    java
    Copy code
    // 例：UserMapper.java
    import org.apache.ibatis.annotations.Mapper;
    import org.apache.ibatis.annotations.Select;

    @Mapper
    public interface UserMapper {
    @Select(“SELECT * FROM user WHERE id = #{id}”)
    User getUserById(Long id);

    // 可以添加其他方法，如插入、更新、删除

    }
    或者，你可以使用 XML 配置 SQL 语句，而不是在接口中使用注解。

    1. 创建 Service 层
      Service 层封装业务逻辑，调用 Mapper 来访问数据库。

    java
    Copy code
    // 例：UserService.java
    import org.springframework.beans.factory.annotation.Autowired;
    import org.springframework.stereotype.Service;

    @Service
    public class UserService {

    private final UserMapper userMapper;

@Autowired
public UserService(UserMapper userMapper) {
    this.userMapper = userMapper;
}

public User getUserById(Long id) {
    return userMapper.getUserById(id);
}

// 其他业务逻辑方法

    }
    4. 创建 Controller
    Controller 层处理 HTTP 请求，并调用 Service 层。

    java
    Copy code
    // 例：UserController.java
    import org.springframework.beans.factory.annotation.Autowired;
    import org.springframework.web.bind.annotation.*;

    @RestController
    @RequestMapping(“/users”)
    public class UserController {

    private final UserService userService;

@Autowired
public UserController(UserService userService) {
    this.userService = userService;
}

@GetMapping("/{id}")
public User getUserById(@PathVariable Long id) {
    return userService.getUserById(id);
}

// 其他RESTful API方法

    }
    5. MyBatis 配置
    在 application.properties 或 application.yml 中配置数据库和 MyBatis 相关设置。

    properties
    Copy code

    spring.datasource.url=jdbc:mysql://localhost:3306/your_db
    spring.datasource.username=dbuser
    spring.datasource.password=dbpass

    mybatis.mapper-locations=classpath:mappers/*.xml
    mybatis.type-aliases-package=com.example.yourapp.model
    注意事项
    XML vs 注解：可以选择使用 XML 文件配置 SQL 语句，或者在 Mapper 接口中使用注解。XML 提供了更高的灵活性。
    事务管理：在 Service 层处理事务。
    异常处理：合理处理异常，尤其是数据库访问异常。
    性能优化：注意 SQL 语句的优化，避免性能瓶颈。
    使用 MyBatis 和 Spring Boot，你可以精确控制 SQL 语句的编写，同时便利地将数据库操作集成到 Spring 应用中。
